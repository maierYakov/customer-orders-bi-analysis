# Описание данных

В проекте используются учебные таблицы `customers` и `orders`.

## Таблица customers

Таблица содержит информацию о клиентах.

| Поле | Описание |
|---|---|
| customer_id | Уникальный идентификатор клиента |
| customer_name | Имя клиента |
| city | Город клиента |
| registration_date | Дата регистрации клиента |

## Таблица orders

Таблица содержит информацию о заказах.

| Поле | Описание |
|---|---|
| order_id | Уникальный идентификатор заказа |
| customer_id | Идентификатор клиента |
| order_date | Дата заказа |
| amount | Сумма заказа |
| status | Статус заказа |

## Возможные статусы заказов

| Статус | Описание |
|---|---|
| paid | Заказ оплачен |
| unpaid | Заказ создан, но не оплачен |
| canceled | Заказ отменён |

## Связь между таблицами

Таблицы `customers` и `orders` связаны через поле `customer_id`.

Один клиент может иметь несколько заказов.  
Один заказ относится к одному клиенту.

## Основные задачи анализа

- Посчитать количество заказов.
- Рассчитать сумму оплаченных заказов.
- Определить средний чек.
- Сравнить выручку по городам.
- Найти активных клиентов.
- Найти клиентов без заказов.
- Разделить заказы на сегменты по размеру оплаты.
- Подготовить данные для BI-отчётности.
