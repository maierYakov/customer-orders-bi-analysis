# Customer Analytics & Retention BI Project

BI-проект по анализу клиентов, заказов, оплат, выручки, повторных покупок и клиентских сегментов с помощью SQL.

## Цель проекта

Проанализировать клиентскую активность, продажи и удержание клиентов, рассчитать ключевые бизнес-метрики и подготовить данные для BI-отчётности.

## Основные задачи

- Изучить структуру данных по клиентам и заказам.
- Подготовить схему таблиц и тестовые данные.
- Объединить данные клиентов и заказов.
- Рассчитать ключевые бизнес-метрики.
- Проанализировать выручку по городам.
- Найти активных клиентов и клиентов без заказов.
- Провести сегментацию заказов по размеру оплаты.
- Выполнить RFM-анализ клиентов.
- Провести retention-анализ повторных покупок.
- Подготовить описание BI-дашборда.
- Сформулировать выводы по результатам анализа.

## Используемые данные

В проекте используются две учебные таблицы:

- `customers` — данные о клиентах;
- `orders` — данные о заказах.

Таблицы связаны по полю `customer_id`.

## Основные метрики

- Общая выручка
- Количество заказов
- Количество оплаченных заказов
- Средний чек
- Доля оплаченных заказов
- Количество клиентов
- Количество активных клиентов
- Выручка по городам
- Повторные покупки
- Клиентские сегменты

## RFM-анализ

В проекте выполнен RFM-анализ клиентов по трём показателям:

- `Recency` — сколько дней прошло с последней покупки;
- `Frequency` — сколько оплаченных заказов сделал клиент;
- `Monetary` — сколько денег принёс клиент.

На основе этих показателей клиенты распределяются по сегментам:

- `best_customers`
- `high_value_customers`
- `new_customers`
- `loyal_customers`
- `at_risk_customers`
- `regular_customers`

## Retention-анализ

Retention-анализ используется для оценки повторных покупок клиентов.

Клиенты разделяются на сегменты:

- `no_paid_orders` — нет оплаченных заказов;
- `one_time_customer` — одна оплаченная покупка;
- `repeat_customer` — две и более оплаченные покупки.
