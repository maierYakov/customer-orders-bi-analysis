# Описание BI-дашборда

Проект можно использовать как основу для BI-дашборда по анализу клиентов, заказов, оплат и удержания.

## Цель дашборда

Показать ключевые показатели клиентской активности, выручки, заказов и повторных покупок в удобном виде для анализа и принятия решений.

## Основные блоки дашборда

### 1. Общие показатели

- Общая выручка
- Количество заказов
- Количество оплаченных заказов
- Средний чек
- Доля оплаченных заказов
- Количество клиентов
- Количество активных клиентов

### 2. Анализ выручки

- Выручка по городам
- Средний чек по городам
- Количество заказов по городам
- Сравнение оплаченных, неоплаченных и отменённых заказов

### 3. Анализ клиентов

- Количество клиентов по городам
- Активные клиенты
- Клиенты без заказов
- Клиенты с повторными покупками
- Клиенты без оплаченных заказов

### 4. RFM-сегментация

В дашборде можно отобразить клиентские сегменты на основе RFM-анализа:

- Лучшие клиенты
- Ценные клиенты
- Новые клиенты
- Лояльные клиенты
- Клиенты в зоне риска
- Обычные клиенты

### 5. Retention-анализ

Retention-блок помогает оценить повторные покупки клиентов.

Основные сегменты:

- Клиенты без оплаченных заказов
- Клиенты с одной покупкой
- Клиенты с повторными покупками

## Возможные визуализации

- Карточки с ключевыми метриками
- Таблица клиентов и заказов
- Столбчатая диаграмма выручки по городам
- Круговая диаграмма по статусам заказов
- Таблица RFM-сегментов
- Таблица retention-сегментов
- График количества заказов по датам

## Польза дашборда

Такой дашборд помогает:

- оценивать общую клиентскую активность;
- находить наиболее ценные клиентские сегменты;
- анализировать выручку по городам;
- отслеживать повторные покупки;
- находить клиентов без заказов;
- готовить данные для управленческой отчётности.

## Итог

Описание дашборда показывает, как SQL-запросы и рассчитанные метрики могут быть использованы для построения BI-отчётности и анализа бизнес-показателей.
